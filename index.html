模板实参推断

形参是T

我们传递这6种实参，实例化后的模板函数func，类型T都推断为int，因为此时是传值，并且顶层const会被忽略（所以传递const int ci，T也推断为int）

形参是T&

当函数参数是模板类型参数的一个普通左值引用时（即，T&），绑定规则告诉我们，只能传递一个左值，所以情况6）会编译错误，而其他情况正常（不要忘了，右值引用是一个左值），实参可以是const类型或非const类型，而根据底层const不会被忽略规则，假如实参是const，则T被推断为const。

当我们传递一个右值引用给模板函数时，类型T推断为int &&，而根据引用折叠规则，实际上实例化的函数为 void func(int& val), 也就是说假如我们在函数体内改变了val的值，会影响到外面的值。

形参是const T&

如果一个函数参数类型是const T&，正常绑定规则告诉我们可以给它传递任何类型实参

  • 一个对象
    • const
    • 非const
      • 因为可以将一个非const对象的引用（或指针）传递给一个const的引用（或指针）形参
  • 纯右值
    • 不要忘了，纯右值可以用来初始化一个常量左值引用

当函数参数本身是const时，T的类型推断的结果不会是一个const类型，因为const已经是函数参数类型的一部分了，它不会再是模板参数类型的一部分。

同样当我们传递一个右值引用给模板函数时，类型T推断为int &&，而根据引用折叠规则，实际上实例化的函数为 void func(const int& val)

形参是T&&

当一个函数参数是一个右值引用时，显然我们可以传递一个右值给它，所以情况6），毫无疑问，类型T被推断为int。

但是情况1-5，显然是一个左值，正常情况下，我们是无法把一个右值引用绑定到左值身上的，但是C++语言指定了一个例外：当将一个左值传给一个参数为右值引用的模板函数（T&&），此时编译器会推断模板类型参数T为实参的左值引用类型（即，T&），同时根据底层引用不会被忽略规则， 情况2和4，T为const int&

所以一个模板函数形参是右值引用的参数，可以给它传递任何类型实参

引用折叠

当形参是T&&， 传递给函数int&的时候，我们可以看到函数的形参val类型是int& &&，也就是引用的引用这种情况，这种情况下，这些引用会形成“折叠”，折叠成左值的引用或者右值的引用。

  • X& &、X& &&、X&& &都被折叠成X&
  • 类型X&& &&被折叠成X&&

完美转发（右值引用+引用折叠+forward）

是什么？

​ 可以把一个函数的实参所有的类型信息全都不变地转发给其他函数，包括实参类型是否是const的以及实参是左值还是右值。

比如当我们用一个左值调用func时，希望func函数体内调用的是左值版print，当用一个右值调用func时，希望右值版的print被调用。

如何实现？

观察模板实参推断4种形参，可知，假如我们需要保存实参的const信息，那形参必须是T& 或者 T&&，同时考虑到T&无法绑定到右值，所以我们只能选用T&&。

然而当用一个右值调用func时（即，func(std::move(x));)，由上述模板实参推断可知，T推断为int，函数被实例化为void func(int && v), 此时v为左值，所以print依然调用左值引用版。

而模板函数forward可以解决这个问题，forward必须通过显示模板实参来调用，forward返回该显式实参类型的右值引用，即forward<T>的返回类型是T&&。

现在func修改为：

  • 当用一个左值调用func时：
    • T推断为int&
    • std::forward<int&> 得到 int&& & ，根据引用折叠规则，即int&， 所以调用左值版本的print
  • 当用一个右值调用func时：
    • T推断为int
    • std::forward 得到 int&& ， 所以调用右值版本的print

所以一个完美转发 = 右值引用类型的模板函数参数+std::forward

tips

  • C++ primer 要求 move和forward，需要使用std::move 和std::forward， 而不要使用using声明，原因详见18.2.3节。

左值和右值

发表于 2022-03-09 分类于 C/C++基础

值类别

自C++11开始，表达式的值分为

  • 左值(lvalue, left value)

  • 将亡值(xvalue, expiring value)

  • 纯右值(pvalue, pure ravlue)

  • 以及两种混合类别：

    • 泛左值(glvalue, generalized lvalue, 左值+将亡值)
    • 右值(rvalue, right value， 纯右值+将亡值)

左值

  • 左值是有实际存储地址的值（具名），可以通过取地址符&获取其地址
  • 常见的左值类型
    • 变量名、函数名以及数据成员名
    • 返回左值引用的函数调用
    • 字符串字面值（如”abc”）

将亡值

  • 将亡值，顾名思义是即将消亡的值，是有临时存放位置（具名），但是马上要销毁的值（可移动）。
  • 是跟右值引用相关的表达式，只能通过以下方式来获得：
    • 返回右值引用T&&的函数返回值
    • std::move的返回值
    • static_cast<T&&>(t);

纯右值

  • 没有存放位置（非具名），且不可移动的值
  • 常见的纯右值类型：
    • 字面值(字符串字面值除外)，例如1，’a’, true等
    • 返回值为非引用的函数的返回值
    • 算术表达式，比如 a+b
    • 逻辑表达式
    • 比较表达式

注意点

  • 字符串字面值是左值，其他字面值都是纯右值，一个很重要的原因，就是可以字符串字面值可以获取地址，这是因为C++将字符串字面值实现为char型数组，实实在在地为每个字符都分配了空间并且允许程序员对其进行操作。如果从存储区的概念来理解，那就是字符串字面值存储在常量区。

引用

在C++11中，包含如下3中引用：

  • 左值引用 int& a
  • 常量左值引用 const int& b
  • 右值引用 int&& c = 10

左值引用和常量左值引用是绑定左值的引用，而右值引用的作用是为了绑定右值。

4.2 虚函数实现模型

发表于 2022-03-05 更新于 2022-03-06 分类于 深度探索C++对象模型

在C++中, 多态表示“以一个基类的指针或引用，寻址出一个子类的意思

虚函数的一般模型：

  • 每一个含有虚函数的类有一个虚函数表，里面包含了这个类里面所有虚函数的地址和该类类型信息， 然后这个类的每个对象有一个虚表指针，指向这个虚函数表。

单继承模型

实现机制

单继承虚函数的实现机制就是虚函数的一般模型，用这种模型来实现虚函数得益于在C++中,虚函数的地址在编译期是可知的，而且这一地址是固定不变的。而且表的大小不会在执行期增大或减小。

虚表中有什么？

一个类的虚函数表中存储 类型信息（通常存储在索引为0的位置）和所有虚函数地址。这些虚函数包括三种：

  • 这个类定义的虚函数，假如基类也定义有这个虚函数，那会改写虚表中相同索引值存储的地址。
  • 继承自基类的虚函数实体，——基类定义有，而这个类却没有定义。直接继承之。
  • 一个纯虚函数实体。用来在虚函数表中占座，有时候也可以当做执行期异常处理函数。

每一个虚函数都被指派一个固定的索引值，这个索引值在整个继承体系中保持前后关联，例如,假如z()在基类虚函数表中的索引值为2，那么在子类的虚函数表的索引值也为2。

虚表的构建步骤

当一个类单继承自有虚函数的基类的时候，将按如下步骤构建虚函数表：

  • 继承基类中声明的虚函数——这些虚函数的实体地址被拷贝到继承类中的虚函数表中相同的slot中。
  • 如果有改写（override）基类的虚函数，那么在1中应将改写（override）的函数实体的地址放入对应的slot中而不是拷贝基类的。
  • 如果有定义新的虚函数，那么将虚函数表扩大一个slot以存放新的函数实体地址。

4.1 各种成员函数的调用方式（普通，static，虚函数）

发表于 2022-02-28 更新于 2022-03-05 分类于 深度探索C++对象模型

c++支持三种类型的成员函数，分别为static,nostatic,virtual。每一种调用方式都不尽相同。

非静态成员函数 （Nonstatic Member Functions）

  • 保证nostatic member function至少必须和一般的nonmember function有相同的效率是C++的设计准则之一。

  • 事实上在c++中非静态成员函数（nostatic member function）与普通函数的调用也确实具有相同的效率。因为编译器内部会将成员函数等价转换为非成员函数，所以本质上非静态成员函数就如同一个普通函数, 如一个非静态成员函数：

    X float Point::X() 就相当于一个普通函数

    float X(Point * const this)

    编译器具体是这样做的:

    • 改写成员函数的签名，使得其可以接受一个额外参数，这个额外参数即是this指针：

    • ```c++
      float Point::X();
      //成员函数X被插入额外参数this
      float Point:: X(Point* const this );
      // 当然如果成员函数是const的，插入的参数类型将为 const Point * const 类型。（指向常量的常量指针）

也就是通过虚函数指针找到虚函数表，然后根据固定的表项找到实际要执行的虚函数。

  • 其中vptr为指向虚函数表的指针，它由编译器产生。vptr也要进行名字修饰，因为一个继承体系可能有多个vptr。

  • 1是虚函数在虚函数表中的索引，通过它关联到虚函数function().

  • 何时发生这种转换？

    • 当通过指针或者引用调用的时候，因为要调用的函数实体无法在编译期决定，必需待到执行期才能获得，
    • 但是当我们通过对象（不是引用，也不是指针）来调用的时候，
      就不会发生，因为在编译器要调用的函数实体已经被决定。此时调用的就是根据指针的静态类型决定的函数。

4.0 名字修饰

发表于 2022-02-28 分类于 深度探索C++对象模型

c++会对一些函数、变量的名字做修改，以达到让这个名字独一无二的目的，实现函数重载，子类的数据成员隐藏同名的父类数据成员，这两种目的。

  • 会对哪些名字进行修改？

    • 普通函数，类成员函数（包括普通成员函数，static成员函数，虚函数）
      • 根据函数的参数列表（参数的个数，参数的类型）+ 函数名， 不包括返回类型
      • 这也是函数重载实现的机制，同名但是实际不同名（因为参数列表不一样）。
    • 类的数据成员
      • 根据类名 + 数据成员名

  • 使用 extern C 会压抑名字修饰
