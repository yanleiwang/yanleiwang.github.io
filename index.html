隐式类型转换

隐式类型转换：自动执行的类型转换（从一种给定类型转换为另一种相关类型）

发生隐式类型转换的情况：

  • 在大多数表达式中，比int类型小的整型值首先提升为int类型
  • 在条件中，非布尔值转换成布尔类型
  • 初始化过中，初始值转换成变量的类型
  • 在赋值语句中，右侧运算对象转换成左侧运算对象的类型
  • 函数调用时

隐式转换的种类:

  • 数组转换成指针： 在大多数用到数组的表达式中，数组会自动转换成指向数组首元素的指针

  • 指针的转换：

    • 0， nullptr能转换成任意指针类型
    • 指向任意非常量的指针能转换成void* （反之，比如void * 转成int* 需要static_cast<int*>）
    • 指向任意对象的指针能转换成const void*
    • 可以把指向子类的指针转换成指向父类的指针，引用也是如此（但是只有public继承才可，其他不行）

  • 转换成布尔类型：

    • 存在一种从算术类型或指针类型向布尔类型自动转换的机制
    • 如果指针或算术类型的值为0，则转换结果是false；否则为true

  • 允许将指向非常量类型的指针转换成指向相应的常量类型的指针，对引用也是如此

  • 类类型的隐式转换

    • 定义了转换构造函数（只有一个参数的构造函数）或者定义了类型转换运算符，就可以执行从一个类隐式转换到另一个类。

    • 只能执行一次类类型的转换，但在这次转换之前or之后可以执行多次以上的转换

算术转换

在算术类型参与的表达式中：

  • 首先，比int类型小的整型值首先提升为int类型

  • 之后如果两个运算对象的类型还是不一致，则小类型的运算对象会转换成较大的类型（可能就发生有符号类型转换成无符号类型了！！！）。

类类型转换

  • 定义了转换构造函数（只有一个参数的构造函数）或者定义了类型转换运算符，就可以执行从一个类隐式转换到另一个类。
  • 只能执行一次类类型的转换，但在这次转换之前or之后可以执行多次以上的转换

子类到基类的转换

用子类初始化或赋值一个基类的时候，实际上是在调用某个构造函数或者赋值运算符函数，只不过因为函数的参数是基类的引用，所以这里发生了子类的引用到基类的引用这样的隐式转换。

实际上并没有子类到基类的隐式转换，只有子类的引用（指针）到基类的引用（指针）这样的隐式转换， 所以如果并没有定义相应的构造函数或赋值运算符函数，那么子类到基类的转换，编译器就会报错。

发表于 2022-06-04 更新于 2022-06-05 分类于 Unix/Linux

管道

  • 管道可以用来在两个相关进程间通信（具有公共祖先的两个进程）。
  • 在管道中数据的传递方向是单向的。管道的一段用于写入，另一端则用于读取
  • 管道其实是一个在内核内存中维护的缓冲器，这个缓冲器的存储能力是有限的。一旦管
    道被填满之后，后续向该管道的写入操作就会被阻塞直到读者从管道中移除了一些数据为止。
  • 管道通常用于两个兄弟进程之间的通信——它们的父进程创建了管道，然后创建两个子进程，然后两个子进程分别关闭读端和写端。关闭的原因是：
    • 确保进程不会耗尽其文件描述符的限制
    • 读取数据的进程需要关闭管道的写端，这样当其他进程完成输出并关闭写端之后，读者就能够看到文件结束（在读完管道中的数据之后）。 否则因为内核知道至少还存在一个管道的写入描述符打开着，即读取进程自己打开着这个写端，则读者也不会看到文件结束。
    • 写入进程关闭其持有的管道的读取描述符是出于不同的原因。当一个进程试图向一个管
      道中写入数据但没有任何进程拥有该管道的打开着的读取描述符时，内核会向写入进程发送一个 SIGPIPE 信号。在默认情况下，这个信号会杀死一个进程。但进程可以捕获或忽略该信号，这样就会导致管道上的 write()操作因 EPIPE 错误（已损坏的管道）而失败。收到 SIGPIPE信号或得到 EPIPE 错误对于标示出管道的状态是有用的，这就是为何需要关闭管道的未使用读取描述符的原因。
  • 只有当所有进程中所有引用一个管道的文件描述符被关闭之后才会销毁该管道以及释放该管道占用的资源以供其他进程复用。

FIFO（命名管道）

从语义上来讲， FIFO 与管道类似，它们两者之间最大的差别在于 FIFO 在文件系统中拥有一个名称，并且其打开方式与打开一个普通文件是一样的。这样就能够将 FIFO 用于非相关进程之间的通信（如客户端和服务器）。

一旦打开了 FIFO，就能在它上面使用与操作管道和其他文件的系统调用一样的 I/O 系统调用了（如 read()、 write()和 close()）。与管道一样， FIFO 也有一个写入端和读取端，并且从管道中读取数据的顺序与写入的顺序是一样的。 FIFO 的名称也由此而来：先入先出。 FIFO 有时候也被称为命名管道。

与管道一样，当所有引用 FIFO 的描述符都被关闭之后，所有未被读取的数据会被丢弃。

Unix/Linux 信号

发表于 2022-05-19 更新于 2022-05-27 分类于 Unix/Linux

基本概念

信号是发生某种事件的通知机制，可以由内核、另一进程或进程自身发送给进程。

信号的传递时机

传递时机指的是调用对应信号处理程序的时机。

信号的产生方式既可以是异步的，也可以是同步的：

当由内核或另一进程调用kill发送信号给进程时，信号是异步产生的。进程无法精确预测异步产生信号的传递时间。 异步信号会在进程从内核态切换到用户态时进行传递，即：

  • 进程在前度超时后，再度获得调度时（即，在一个时间片的开始处）。
  • 系统调用完成时（信号的传递可能引起正在阻塞的系统调用过早完成）。

因进程自身执行代码而直接产生的信号则属于是同步产生的，例如，执行了一个引发硬件异常的指令，或者去调用raise、kill向自身发送信号。同步生成的信号，会立即传递。、

其他特点

  • 待处理信号被阻塞：

    现代signal()和sigaction()的默认语义会在执行处理函数时将当前信号自动添加到进程信号掩码中，从处理函数返回时，自动将当前信号从掩码中删除。比如，当前正在运行SIGINT处理函数，此时另一个SIGINT信号产生，那么这个新的SIGINT将变成待处理，直到处理程序返回。

  • 待处理信号不会排队等待：

    标准信号至多只有一个待处理信号。比如几乎同时n个SIGINT信号产生，那么待处理信号只会有一个SIGINT，其他的SIGINT会被丢弃。（待处理信号集基本都是用位图来实现，所以其实很好理解）

  • 慢速系统调用可以被中断：

    如果进程在执行一个慢速系统调用（比如read(), write() 终端时），而阻塞期间捕捉到一个信号，那么该系统调用可能就被中断不再继续执行。（sigaction()中如果采用 SA_RESTART 标志来注册处理器函数，则在遭到中断时是可以自动重启的 ）

  • 正在执行一个信号处理函数时，也可能会传递另一个信号过来

    假设，当前待处理信号包含信号A、B。先执行信号A的处理函数，执行过程中从内核态返回到用户态，此时就会跳去执行信号B的处理函数，从信号B处理函数返回时，继续执行信号A的处理函数。

  • 除了返回之外，信号处理器函数的终止还存在多种其他方法，其中包括：调用_exit()，发
    送信号来终止进程（ kill()、 raise()或 abort()），或者执行非本地跳转。借助于 sigsetjmp()和
    siglongjmp()，可以在执行非本地跳转时为程序提供处理信号掩码的显式控制手段。

应用场景

尽管可以将信号视为 IPC 的方式之一，但诸多制约因素令其常常无法胜任这一目的，其中包括信号的异步本质、不对信号进行排队处理的事实，以及较低的传递带宽。信号更为常见的应用场景是用于进程同步，或是各种其他目的（比如，事件通知、作业控制以及定时器到期）。

异步信号安全函数

考虑到异步信号可以从任何时刻传递、慢速系统调用可以被中断、正在执行一个信号处理函数时，也可能会传递另一个信号过来等特点。只有只调用异步信号安全函数的信号处理函数才是安全的，。

异步信号安全函数包括：不可中断函数和可重入函数。

不过调用异步信号安全函数依然可能会导致信号处理器函数不可重入，因为异步信号安全函数可能会更新errno，会覆盖之前由主程序调用函数时所设置的 errno 值。所以写信号处理函数时需要在其入口处保存 errno 值，并在其出口处恢复 errno 的旧有值

可重入函数

如果一个函数可以在任意时刻被中断，转头去调用另一段代码，这段代码又调用了这个函数，最终结果不会出错（和依次调用该函数的结果是一致的），那么这个函数就是可重入函数。

简而言之，可以重入执行的函数是可重入函数。

要编写可重入函数，必须：

  1. 不要使用全局或static变量
  2. 不要调用其他非可重入函数

只会更新本地变量的函数肯定是可重入函数。更新全局变量或静态数据结构的函数就是不可重入的。（只用到本地变量的函数肯定是可重入的。） 比如malloc()和 free()就维护有一个针对空闲内存块的链表，用于从堆中重新分配内存。如果主程序在调用 malloc()期间为一个同样调用 malloc()的信号处理器函数所中断，那么该链表可能会遭到破坏。因此，malloc()函数族以及使用它们的其他库函数都是不可重入的。

所有的系统调用都是可重入的，一部分库函数是可重入的（存疑）

线程安全函数

在多个线程里，可以同时安全调用的函数，是线程安全函数。意味着会同步对共享资源的访问（比如加锁）

可重入强调在单线程执行时重新进入同一个函数，结果仍然是安全的。而线程安全，是多个线程可以同时调用。可重入函数不一定是线程安全函数，而线程安全函数也不一定是可重入函数。

  • 线程安全，但不可重入：

    函数在它的函数体内部访问共享资源使用了加锁、解锁操作，所以它是线程安全的，但是却不可重入。因为若该函数运行到已经执行加锁但未执行解锁时被中断，中断处理函数又去调用该函数，那么在加锁的时候就会导致死锁，使得中断处理函数永远无法返回。

常用函数

signal

sigaction

有些系统上的signal()还是旧语义(执行信号处理函数时会重置信号处理函数为SIG_DFL, 并且不会屏蔽当前信号)， 所以出于可移植性考虑，最好使用 sigaction()来建立信号处理器函数。

如果在调用 sigaction()时设置了 SA_SIGINFO 标志，那么所创建的信号处理器函数就能接收信号的附加信息。 siginfo_t 结构提供了这些信息，其地址则传递给信号处理器作为参数。

如果信号处理器函数中断了阻塞的系统调用，系统调用会产生 EINTR 错误。利用这种特性，就可以为阻塞的系统调用设置一个定时器。如果有意，可以手动重启遭到中断的系统调用。另外，在调用 sigaction()创建信号处理器函数时，如果设置了 SA_RESTART 标志，那么大部分（但非全部）系统调用都将会自动重启。

与 signal()相比， sigaction()系统调用在设置信号处置方面提供了更多控制，且更具灵活性。
首先，可以指定一组调用处理器函数时将阻塞的额外信号。此外，可以使用各种标志来控制调
用信号处理器时所发生的行为。例如，启用某些标志即可选择旧有的不可靠信号语义（不阻塞
引发处理器调用的信号，在调用信号处理器之前就将信号处置重置为默认值）。

kill和raise

  • kill不是用来杀死进程的，是用来发送信号的，只不过大多数信号有杀死进程的作用
  • raise(signum) 等价与kill(getpid(), signum);

alarm和pause

  • 使用alarm函数可以设置一个定时器，在seconds秒后定时器超时，产生SIGALRM信号
  • 调用 pause()将暂停进程的执行，直至被一个信号中断为止

信号集

  • sigset_t 信号集类型
  • sigemptyset() 将一个信号集置为空集
  • sigfillset() 将一个信号集置为全集
  • sigaddset() 将一个信号加入信号集
  • sigdelset() 将一个信号移除信号集
  • sigismember()

sigprocmask

  • 每个进程都具有一个信号掩码，代表当前传递遭到阻塞的一组信号。使用 sigprocmask()
    可从信号掩码中添加或者移除信号。

sigpending

进程能够使用 sigpending()系统调用来获取等待信号集（用以描述多个不同信号的
数据结构）。

sigsuspend

sigsuspend()系统调用在自动修改进程信号掩码的同时，还将挂起进程的执行直到信号到
达，且二者属于同一原子操作。

others

可以使用 sigwaitinfo()和 sigtimedwait()来同步等待一个信号。这省去了对信号处理器的
设计和编码工作。对于以等待信号的传递为唯一目的的程序而言，使用信号处理器纯属多此
一举。 像 sigwaitinfo()和 sigtimedwait()一样，可以使用 Linux 特有的 signalfd()系统调用来同步等
待一个信号。这一接口的独特之处在于可以通过文件描述符来读取信号。还可以使用 select()、
poll()和 epoll 来对其进行监控。

实时信号

实时信号是 POSIX 对原始信号模型的扩展，不同之处包括对实时信号进行队列化管理，
具有特定的传递顺序，并且还可以伴随少量数据一同发送。设计实时信号，意在供应用程序
自定义使用。实时信号的发送使用 sigqueue()系统调用，并且还向信号处理器函数提供了一
个附加参数（ siginfo_t 结构），以便其获得信号的伴随数据，以及发送进程的进程 ID 和实际用
户 ID。

Unix/Linux 文件系统

文件系统内部结构

3层抽象

硬盘实际上是由一些磁性盘片组成的一个设备，而文件系统是对该设备的一种多层次的抽象。

第一层抽象 从磁盘到分区

可以把一个硬盘分成 一个或多个分区，每个分区都包含一个文件系统，可以看做是一个独立的硬盘。

第二层抽象 从磁盘到块序列

一个硬盘由多个磁性盘片组成，每个盘片的表面可以被分为很多同心圆，这些同心圆称为磁道，每个磁道又进一步被分成扇区，每个扇区可以存储一定字节数的数据。例如常见的每个扇区是512字节。扇区是硬盘的基本存储单元。

因此我们可以把硬盘视为一个块(扇区)序列。再结合第一层抽象：一个分区也由多个扇区组合而成。

第三层抽象 从块序列到3个区域的划分

一个分区（文件系统），主要以下3个区域：

  • 超级块

    • 存放文件系统本身的结构信息，比如文件系统数据块大小，文件系统类型，指向根目录的指针，inode区位图和数据区位图（APUE并没有把这两个位图划分到超级块，详见总结）

  • inode节点表

    • inode节点，中文名为索引结点，包含文件的相关基本信息，比如

      • 文件所有者，组，权限，修改日期，文件大小等等
      • 文件数据在数据区的位置
      • 硬链接数
      • 相关的操作方法列表（read， write等等）

      除了文件名之外的所有文件信息，都存储在inode之中。所有文件都有一个对应的 inode 结构， 所以两个不同的文件名，如果inode号一致，则实际指向的是同一个文件（可以用ls -li命令来查看）

  • 数据区

    • 文件的内容保存在这个区域，磁盘上所有的块的大小是相同的。如果文件包含了超过一个块的内容，则文件内容会存放在多个磁盘块中。

  • 磁盘读写的最小单位是扇区，扇区的大小只有 512B 大小，很明显，如果每次读写都以这么小为单位，那这读写的效率会非常低。所以，文件系统把多个扇区组成了一个逻辑块，每次读写的最小单位就是逻辑块（数据块），大小可以由系统管理员在创建文件系统时决定，Linux 中默认逻辑块大小为 4KB，也就是一次性读写 8 个扇区，这将大大提高了磁盘的读写的效率

总结

可以把一个磁盘分成一个或多个分区，每个分区是一个文件系统，每一个分区里面包含：

  • 超级块
    • 存放文件系统本身的结构信息
  • inode节点位图，数据块位图
    • 每个二进制位对应一个盘块。“0”代表盘块空闲， “1”代表盘块已分配
  • inode节点表
    • 表项是inode节点，一个固定长度的记录项，包含文件除文件名和具体数据以外的所有信息。所有文件都有一个对应的 inode 节点。
  • 数据区
    • 存放文件的数据

文件系统的实现

目录文件

目录文件的数据实际是当前目录下各个文件名和对应的inode号（包括当前目录和父目录文件，分别用. 和.. 表示），除此之外与普通的数据文件并没有什么差别，同样具有inode节点（可能inode节点里存储的操作方法列表不一样）。即目录文件内部数据如下：

创建一个文件的过程

  1. 存储属性
    • 内核先找到一个空的inode节点，把文件的信息记录其中
  2. 存储数据
    • 从数据块位图中找到空闲块，把数据复制到空闲块中
  3. 记录分配情况
    • 把文件所占的数据块序号列表写进inode节点中
  4. 添加文件名到目录
    • 内核把文件名和inode编号写进当前目录的目录文件中

打开一个文件的过程

  1. 获取一个空闲的文件描述符
  2. 在目录中寻找文件名，拿到对应的inode节点号
    • 要想拿到inode节点号，就需要读取对应的目录文件
      • 假如文件路径是以绝对路径的方式给出的，因为我们知道根目录的inode节点号（超级块中存放着指向根目录的指针），所以可以一层一层的处理下来，打开对应的目录文件
      • 假如文件路径是以相对路径的方式给出的，当前目录的目录文件也缓存在内存中，所以同理我们可以打开对应的目录文件
  3. 定位对应inode节点号，并读取其内容，生成对应的文件表（下面会介绍）
  4. 把文件描述符与文件表关联起来
  5. 返回文件描述符

读写一个文件的过程

  • 读
    • 根据文件描述符获取到文件对应的inode节点，然后得到对应的数据块编号列表，把数据从磁盘复制到内核缓冲区，再把数据从内核缓冲区复制到用户空间。
  • 写
    • 同理

硬链接和符号链接

有时候我们希望给某个文件取个别名，那么在 Linux 中可以通过硬链接和符号链接的方式来实现，它们都是比较特殊的文件，但是实现方式也是不相同的。

硬链接

硬链接是共享inode节点的，也就是指向同一个 inode，但是inode 是不可能跨越文件系统的，每个文件系统都有各自的 inode 数据结构和列表，所以硬链接是不可用于跨文件系统的。由于多个硬链接指向一个 inode，那么只有删除文件的所有硬链接时，系统才会彻底删除该文件。

符号链接

符号链接相当于重新创建一个文件，这个文件有独立的 inode，但是这个文件的数据存的就是另外一个文件的路径，所以符号链接是可以跨文件系统的，甚至目标文件被删除了，链接文件还是在的，只不过指向的文件找不到了而已。

而很多函数（比如read，write）访问符号链接的时候，函数内部会跟随符号链接到达它所链接的文件，实际上相当于处理另外一个文件，而有些函数（比如lstat， lchown）是不跟随符号链接的，也就是处理的是符号链接这个文件本身。

磁盘的结构 （不是很重要）

磁盘的表面由一些磁性物质组成，可以用这些磁性物质来记录二进制数据，这个圆圆的东西就是盘片，磁头臂带动磁头进行转动，可以去到不同的磁道上面，磁道的概念如下：

磁盘的盘面被划分成一个个磁道，这样的一个“圈”就是一个磁道， 一个磁道（一圈）又被划分成一个个扇区，每个扇区就是一个“磁盘块”。

各个扇区存放的数据量相同(如1KB)，最内侧磁道上的扇区面积最小，因此其数据密度最大。

读取数据
